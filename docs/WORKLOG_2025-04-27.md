## 关键发现和问题解决

### 问题 #1：Pydantic 赋值不验证
**现象**：`state.iteration = 5` 不报错（超出 le=3 限制）

**原因**：Pydantic v2 默认只在创建实例时验证

**解决**：设置 `model_config = ConfigDict(validate_assignment=True)`

### 问题 #2：Pydantic 对象不支持 .get() 方法
**现象**：`state.get('messages', [])` 报 AttributeError

**原因**：Pydantic BaseModel 不是字典

**解决**：
```python
if isinstance(state, dict):
    messages = state.get('messages', [])
else:
    messages = state.messages
```

### 问题 #3：stream() 只返回部分更新
**现象**：使用 stream() 看起来 messages 没有累加

**原因**：stream() 返回节点返回值（部分更新），不是合并后的状态

**解决**：使用 `invoke()` 获取最终完整状态

## 重要决策

### 决策 #1：推荐使用 Pydantic 版本
**理由**：
- 运行时验证防止无效数据
- 默认值机制简化代码
- 字段约束保障数据质量
- LangGraph 完全兼容

### 决策 #2：Annotated 使用建议
**累加字段（需要 Annotated）**：
- messages, logs, metrics, events

**覆盖字段（不需要 Annotated）**：
- sources, analyses, articles
- 单值字段：count, status, iteration

**手动累加字段**：
- 复杂对象：cost_tracker, metadata

### 决策 #3：最佳实践
1. 设置 `validate_assignment=True`
2. 统一处理 state 类型（dict 或 BaseModel）
3. 使用 invoke() 获取最终状态
4. 在节点内部打印当前状态验证
